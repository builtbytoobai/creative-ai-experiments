# Claude Instructions

This repository is part of the builtbytoobai AI + creative tech portfolio.

## Working style

- Work in small, focused changes.
- Always propose a short plan before editing files.
- Implement one meaningful change at a time.
- Do not modify unrelated files.
- Do not expose secrets or API keys.
- Do not commit `.env`, `outputs/`, generated files, or local-only files.
- Update README when user-facing behavior changes.
- Update roadmap files when project direction changes.
- Prefer clear architecture over over-engineering.

## Git workflow

Use this workflow:

1. Create or use a feature branch.
2. Make one focused change.
3. Run relevant commands/tests.
4. Summarize files changed.
5. Suggest a clear commit message.
6. Wait for user approval before committing.

## Review checklist

Before finalizing a task, check:

- Does the change solve the requested task?
- Are unrelated files untouched?
- Are secrets protected?
- Is the README updated if needed?
- Is the code readable and maintainable?
